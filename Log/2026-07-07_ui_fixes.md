# 2026-07-07 UI不具合修正・z-index調整・入力検証 実施ログ

## 実施内容

本日、以下のバグ修正およびUX改善、インラインスタイルのCSS移行を実施しました。

### 1. z-index階層の整理とトーストの最前面化
- **z-indexの階層化**: `--z-footer`, `--z-recording`, `--z-overlay-slate`, `--z-overlay-director`, `--z-overlay-tally`, `--z-marker-flash`, `--z-guide` などのCSS変数を `:root` に定義し、重なり順を一括管理しました。
- **トーストの最前面化**: `Toaster` の `containerStyle` で `zIndex: 10500` を指定し、タリーやディレクターなどのどのフルスクリーンオーバーレイよりも前面にトーストが表示されるようにしました。
- **デッドCSS削除**: 使用されていなかった `.toast-container` および `.toast` のCSSルールを削除しました。

### 2. フルスクリーンオーバーレイの相互排他
- **相互排他**: タリー・ディレクターパネル・ビジュアルスレートのいずれかを開いた際、他の2つのパネルが自動的に閉じるように onClick イベントハンドラーおよびタップイベントを修正しました。これにより、同時にオーバーレイが開いて操作不能になる不具合を解消しました。

### 3. ビジュアルスレートの表示はみだし解消とレスポンシブ対応
- **はみだし解消**: 狭い画面幅（320px〜375px）の端末でも「REEL / SCENE / TAKE」の情報が横にはみ出さないよう、`flex-wrap: wrap` を適用し、`font-size` に `clamp(...)` を指定しました。
- **インラインスタイルのCSS化**: スレート内のインラインスタイルを、新規追加したCSSクラス（`.slate-tap-area`, `.slate-content`, `.slate-metadata-row`, `.slate-close-btn`）に移行しました。

### 4. テキスト切り詰め・あふれ対策
- **切り詰め(ellipsis)適用**: 
  - `.p2p-status-mini`
  - `.client-id` / `.client-stats`
  - `.tally-header-id`
  - `.director-cam-label-input`
  - `.marker-flash`
  - `.vu-label`
  に `white-space: nowrap; overflow: hidden; text-overflow: ellipsis;` を追加し、動的テキストが長くなっても折返しやレイアウト崩れが起きないようにしました。
- **クライアント一覧の高さ制限**: `.clients-list-section` に `max-height: 40vh; overflow-y: auto;` を追加し、接続数が多くなった際にページが縦に伸びすぎないように制限しました。
- **100dvhの適用**: デスクトップの `.marker-scroll` の高さを `100vh` から `100dvh` に変更し、モバイル等の動的なツールバーによる影響を排除しました。

### 5. インラインスタイルのCSS化
- **インラインスタイルの移行**: マーカー行（2段構造）、TCオフセット行、ユーザービッツ行、VuMeterなどのインラインスタイルを `App.css` のクラスへ移行しました。
- **VuMeterのレイアウトシフト防止**: クリップバッジ（`CLIP`）を常時レンダリングし、非クリップ時は `visibility: hidden`（`.vu-clip-badge.hidden`）で隠すようにしました。これにより、バッジの有無によってメーターの幅がガタつく（レイアウトシフト）現象を修正しました。

### 6. 入力検証・UX改善
- **入力文字数制限（maxLength）**: 
  - 手動TC入力、マスターSTART TC入力：`maxLength={11}`
  - マーカーコメント：`maxLength={100}`
  - ターゲットID入力：`maxLength={8}`
- **P2PクライアントID自動大文字化**: ID入力時に自動的に大文字に変換されるようにしました。
- **クリップボードコピー例外処理**: コピー失敗時にもエラー用トーストが表示されるように `.then().catch()` で実装しました。
- **多言語対応(i18n)**: ハードコードされていた日本語テキスト（`一括操作`, `操作ログ`, `明`, `中`, `暗`, `ディレクターと接続中` 等）を `tr()` 関数を使用した多言語（en/ja）翻訳に対応させました。
