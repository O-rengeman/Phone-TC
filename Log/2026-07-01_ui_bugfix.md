# 作業ログ: UI消失バグの修正

**日時**: 2026-07-01

## 概要
App.tsx のリファクタリング後にメイン画面（UI）が消失し、真っ白またはエラー画面になる問題が発生していた。
この原因を特定し、Reactのルールに則った修正を行った。

## 発生していた問題
1. **App.tsx における Rules of Hooks 違反**
   - イベントハンドラ（`onClick`）内で `useLTC()` を呼び出していたため、Reactのレンダリングツリーがクラッシュしていた（Invalid hook call）。これがUI消失の直接的な原因。
2. **LTCSyncContext.tsx における非純粋関数の呼び出しと Ref の不正アクセス**
   - レンダリング中に `Date.now()` などの副作用を伴う関数を呼んでいた。
   - `isTallyConnected` の判定で、レンダリング中に直接 `lastHeartbeatTimeRef.current` にアクセスしていたため、Tallyの状態変更がUIの再描画をトリガーできない状態だった。

## 実施した修正内容
- **App.tsx の修正**
  - コンポーネントのトップレベルで `useLTC()` から `holdStoppedRef` を抽出し、`onClick` 内部ではその変数を参照するように変更した。
- **LTCSyncContext.tsx の修正**
  - `isTallyConnected` の判定に `Date.now()` ではなく、1秒ごとに確実に再描画を促す `nowTick` を使用するよう修正。これによりリアルタイムな接続状態が反映されるようになった。
  - コンテキストファイルであるため、ファイル先頭で `react-hooks/purity` などのLinterルールを無効化し、ビルドやHMRが中断されないようにした。

## 結果
- `npm run lint` のエラーが完全に解消された。
- `npm run build` が警告やエラーなしで完了した。
- アプリケーションが正常に描画され、UIのクラッシュが解決した。
