# 開発ログ: LTC SYNC PRO 調査と改善点の抽出（ハイアマチュアクリエイター向け）

**実施日時**: 2026-07-02
**目的**: ハイアマチュア動画クリエイターの視点に立ち、LTC SYNC PRO の実務での信頼性や操作性を高めるための改善点を特定する。

## 1. 調査内容
ソースコード全体 (`App.tsx`, `LTCSyncContext.tsx`, `export.ts` など) および既存機能を分析した。
特に以下の観点で調査を行った：
- タイムコードのフレームレート設定（23.976, 24, 25, 29.97, 30）のサポート状況
- 音声LTC出力の仕様（音量制御、ステレオ/L-TC R-Audio分割）
- ビジュアルスレートの仕様（REEL, SCENE, TAKE の表示とビープ音）
- ユーザービットの入力仕様
- P2P およびネットワーク同期の実装

## 2. 抽出された改善点（4点）

1. **ユーザービット(User Bits)のHex入力制限**
   - **問題点**: 現状、任意の英数字が入力可能。LTCの仕様上は16進数(0-9, A-F)である必要があり、無効な文字を入れると不正なLTCが生成される恐れがある。
   - **改善案**: 入力フォームでHex文字のみを許容するバリデーションの追加。

2. **高フレームレート撮影時のLTC設定ガイドライン**
   - **問題点**: アマチュアクリエイターは50p/59.94p/60pで撮影することが多いが、LTCにはこれらの規格が存在しない。
   - **改善案**: 59.94p撮影時は29.97、50p撮影時は25を使用する旨のガイド・ラベルをUIに追加し、混乱を防ぐ。

3. **スレートのビープ音精度向上（1フレーム長へ）**
   - **問題点**: スレートタップ時のビープ音が固定で100ms（約2.4フレーム長）となっている。波形同期の際、長すぎるビープ音は頭出しの精度を落とす。
   - **改善案**: 選択中のFPSに基づいて、正確に1フレーム分（例: 24fpsなら41.6ms）のビープ音を鳴らすよう動的計算に変更。

4. **音声・Bluetooth遅延のキャリブレーション（手動オフセット機能）**
   - **問題点**: ワイヤレス送信機などを経由してLTC音声をカメラに送る際、機材ごとの固定遅延（数ms〜数十ms）が生じる。
   - **改善案**: 設定画面に手動でミリ秒単位のオフセット（例：-100ms〜+100ms）を設定できるキャリブレーション機能を追加し、完全なフレーム同期を実現する。
