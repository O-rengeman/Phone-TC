# 🛠 長押し停止の直後に開始が誤発火する不具合を修正

- **日付**: 2026-06-29
- **種別**: 修正（バグ）
- **主な対象ファイル**: `src/App.tsx`

## 症状

「長押しで停止」した後、指を離した瞬間に**開始ボタンが押されてしまう**（停止→即再開）。

## 原因

長押しが完了すると `isRunning` が false になる。その状態で指を離すと、ブラウザが**末尾の `click` イベント（trailing click）**を発火し、メインボタンの `onClick` が `!isRunning` 判定で START を呼んでしまっていた（同じ操作の指離しが再開を誘発）。

## 修正

- 長押し停止が完了した時点で `holdStoppedRef = true` を立て、直後に来る `click` を**1回だけ無視**して消費する（その後フラグはクリア）。長押し時間が長い場合でも、指離しの `click` で確実に消費されるためタイマーに依存しない。
- ポインタハンドラを整理：`onPointerDown` は常時バインドし内部で `isRunning` を判定、`onPointerUp` / `onPointerLeave` は常に `cancelStopHold`（保持中でなければ無害なno-op）。`onContextMenu` は常時 preventDefault で長押し時のコンテキストメニュー/選択を抑制。

これにより、長押し停止→指離しでは再開せず、別途タップしたときだけ開始します。短いタップ（保持未完了）は従来どおり何も起きません。

## 影響・検証

- `tsc -b` 通過、ESLint クリーン、テスト **86件合格**、Viteビルド **64モジュール成功**。
- 実機（タッチ）での長押し停止→指離しで再開しないことの確認を推奨。
