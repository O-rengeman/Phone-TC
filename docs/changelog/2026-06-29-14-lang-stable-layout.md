# 🛠 言語切替でレイアウトが変わらないように固定

- **日付**: 2026-06-29
- **種別**: 修正（UX）
- **主な対象ファイル**: `src/App.css`

## 背景

日本語/英語を切り替えると文字長が変わるため、言語トグルの幅・状態ピルの幅・各ラベルの折返し行数が変化し、ヘッダーやパネルがガタつく/高さが変わる事象があった。「切替してもレイアウトが全く変わらない」ことが目標。

## 変更内容

footprintが文字内容で決まる要素を**言語非依存に固定**（CSSのみ、末尾追記）。

- **言語トグル**を固定幅（`width:50px`・中央寄せ・nowrap）に。`日本語`↔`EN` で幅が動かない。
- **状態ピル**に `min-width:108px` ＋中央寄せ＋nowrap。各状態の EN/JA が同じ箱に収まり、ヘッダーが横にずれない。
- **タブ/各セクションラベル/tool-card・marker-header のラベル**を `white-space:nowrap`（必要時 `ellipsis`）に。言語で折返し行数が変わらず、**セクション高さが一定**＝縦レイアウトが不変。
- **翻訳テキストのボタン/バッジ**（メインボタン文言、FPSピル、QUICKマーク、P2Pボタン、同期バー、要再同期、ガイドのタイトル/ボタン 等）を1行固定。ボックスはレイアウト側で決まるため、1行維持で高さが一定。

## 影響・検証

- Viteビルド **64モジュール成功**、テスト **86件合格**。
- CSSのみ・既存ルールは温存（同詳細度＋後勝ちの上書き）。データ表示（FPS値・同期モード・残量%・TC）は元々言語非依存。

## 補足

ガイドモーダルの本文（長文ステップ）は言語で行数が変わるが、オーバーレイ（スクロール可・ビューポート基準の高さ）のため背後の本体レイアウトには影響しない。
