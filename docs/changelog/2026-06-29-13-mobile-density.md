# ♻️ モバイルUIのスペース効率改善

- **日付**: 2026-06-29
- **種別**: 変更（UX / レイアウト）
- **主な対象ファイル**: `src/App.css`

## 背景

モバイル表示は各コントロール間の縦ギャップ（`.tab-pane` 16px）、TC下の余白（`.info-strip-pro` 下20px）、QUICKマークボタンの過大な高さ（padding 24px）など、余白が大きく1画面に収まる情報が少なかった。

## 変更内容

`.mobile-view` スコープの密度向上ブロックを末尾追記（モバイル限定・デスクトップ非影響）。

- ヘッダー `14/20` → **10/14**、タブバー `10px` → **8px**、`.tab-content` `12px` → **10px**、`.tab-pane` gap `16px` → **10px**。
- ヒーローのTCカードを `4/1.5` → **`4/1.3`**（やや低くして縦の実スペースを確保。TCは幅オートフィットのため視認性は維持）。`.info-strip-pro` の下マージン `20px` → **8px**、情報ラベルを微縮小。
- コントロール群を圧縮：`.control-section` gap `8` → **6**、`.btn-pill`/`.volume-row`/`.sync-toggle-pro` の padding を縮小。
- **QUICKマークボタンの padding `24px` → `13px`**（最大の無駄を解消）、グリッド gap/余白を縮小。
- マーカー周りのchromeを圧縮し、空いた分**一覧の高さ `150px` → `200px`**（より多くのテイクが一覧表示）。
- フッターの padding を圧縮（セーフエリアは維持）。

## 影響・検証

- Viteビルド **64モジュール成功**、テスト **86件合格**。
- CSSのみ・`.mobile-view` スコープのため、デスクトップ表示とロジックは不変。タップ領域は実用的なサイズを維持。

## 補足

実機（縦持ち）で、設定〜マーカーまでが少ないスクロールで収まること、QUICKマークボタンが過大でないことを目視確認するのが望ましい。
