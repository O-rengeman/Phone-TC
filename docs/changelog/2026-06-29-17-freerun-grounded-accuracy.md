# ✨ 任意TC開始(FREE)モード追加＋精度表示を根拠ある実測値のみに

- **日付**: 2026-06-29
- **種別**: 機能追加 / 変更
- **主な対象ファイル**: `src/App.tsx`, `src/utils/i18n.ts`, `src/App.css`

## 1. 任意のTC時間から始める「FREE」モード

- 同期方式に **`freerun`（FREE）** を追加（SYSTEM / NETWORK / P2P / FREE）。
- FRE選択時、SYNCエリアに **「開始タイムコード」入力欄**（`HH:MM:SS:FF`）を表示。
- START時、**NTPやP2Pを使わず端末クロックでその開始TCから連続再生（free-run）** する。内部的には既存の `engine.setManualTimecode()` を利用（P2Pマスターの手動開始と同じ仕組みを単独モード化）。
- ネットワーク同期の定期再同期/精度パネルは `network` モード限定のため、FREEでは作動しない（クリーン）。

## 2. 精度表示を「根拠のある実測値」だけに

ご要望に沿って、**推定（estimate）系の表示を撤去**しました。

- 削除：推定ドリフト（±ms / フレーム）、クロック誤差（ppm、特に未測定時の `~30ppm (est.)` 仮定値）、推定ベースの ACCURACY 信頼度バッジ（HIGH/MEDIUM/LOW）、MAIN同期バーの信頼度バッジ。
- 残す（すべて実測に基づく事実）：
  - **最終同期からの経過時間**（NTP同期からの実経過）。
  - **NTP Latency / Offset**（直近同期で実測した値。`sync-detail` 行に表示）。
  - **再同期推奨**：推定ドリフトではなく **経過1時間** という事実ベースのしきい値で表示。

これにより、画面上の精度情報は計測値のみとなり、推測の数値は出さなくなりました。

## 影響・検証

- `tsc -b` 通過、ESLint クリーン、テスト **86件合格**、Viteビルド **64モジュール成功**。
- 言語不変は維持（FREEラベルもトグルは flex:1 等幅、開始TC欄は固定高）。
- `DriftMonitor` 自体は最終同期時刻の追跡に引き続き使用（推定値はUIに出さないだけ）。

## 補足

開始TCの書式が不正な場合は開始時に弾かれ、エラートーストを表示します（既存の手動TC入力と同じ挙動）。
