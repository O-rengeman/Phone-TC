  - 変更 `src/utils/PeerSync.ts`, `src/utils/PeerSync.test.ts`

## 背景

3つの同期モード（SYSTEM / NETWORK / P2P）の信頼性を高め、回帰をテストで担保する。

## 変更内容

- **TimeSync（NTP型ネットワーク同期）**:
  - 複数の時刻サーバーを順に試行し、RTTの半分でレイテンシ補正してオフセットを推定。
  - 1時間TTLの**キャッシュ**を保持し、全サーバー失敗時はキャッシュへフォールバック。
  - 4秒のタイムアウト（`AbortController`）でスタックを防止。低レイテンシ(<50ms)サンプル取得で早期確定。
- **PeerSync（P2P / PeerJS）**:
  - マスター/クライアント間の sync-request/response によるTC配布とドリフト補正のメッセージング。
- 両モジュールに対するユニットテストを拡充（TimeSync 8件 / PeerSync 15件）。

## 影響・検証

- 全体テスト 71件合格。
- 本変更は後続の DriftMonitor（精度可視化）の土台でもある（再同期のたびにオフセットを `DriftMonitor.addSync` へ供給）。

## 今後の候補

- NTP同期の複数サンプル+中央値化、サーバー多重化による精度向上。
