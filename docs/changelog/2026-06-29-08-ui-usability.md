# ✨ UI 改良（メイン操作の明確化 / TC視認性・録画中表示）

- **日付**: 2026-06-29
- **種別**: 変更（UX改善）
- **主な対象ファイル**: `src/App.tsx`, `src/App.css`

## 背景

現場で「迷わず操作でき、離れても読める」ことを狙ったUI改善。優先領域は (1) メイン操作の明確化、(2) タイムコードの視認性と録画中の状態表示。

## 変更内容

### TC視認性・録画中表示（キャンバス描画）
- タイムコードを**カード幅の約90%にオートフィット**（`measureText` で実測しスケール）。画面サイズ・縦横問わず最大限大きく表示。
- 文字色を純白 `#ffffff` に上げ、コントラスト強化。LTC出力中は赤系グロー。
- 出力中はキャンバス左上に**パルスする赤い ● + "REC"** インジケーターを描画。

### メイン操作の明確化
- ヘッダーに**常時表示のステータスレール**を追加：状態ピル（READY / SYNCING / LTC OUT、状態色＋ドット）＋ 現在の **FPS** と **同期モード**。タブを切り替えなくても主要状態が分かる。
- フッターの主ボタンを**状態色で大型化**（`min-height:60px`）：
  - 待機/再開 = 緑(START/RESUME)、準備中 = 黄(PREP...)、出力中 = 赤(STOP)。
  - 誤操作防止のため、出力中は明確に赤で「STOP」と判別可能に。
- 重複していた `.running .btn-icon` ルールを整理。

## 影響・検証

- `tsc -b` 通過、ESLint クリーン、テスト **71件合格**、Viteビルド **62モジュール成功**。
- ロジック変更なし（表示・スタイルのみ）。既存機能への破壊的変更なし。
