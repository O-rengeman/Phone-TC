# ✨ バッテリー残量/可動時間表示・マーカー打点フィードバック

- **日付**: 2026-06-29
- **種別**: 機能追加（UX改善）
- **主な対象ファイル**: `src/utils/battery.ts`(新規), `src/utils/battery.test.ts`(新規), `src/App.tsx`, `src/App.css`

## 背景

長回しで「あと何時間持つか」が見えない不安、そして「マーカーが効いたか分からない」迷い——どちらも現場の実害に直結する。

## 変更内容

### バッテリー残量 / 推定可動時間
- 新規 `battery.ts`：放電サンプル(level, time)から **0%到達までの推定分数** を線形外挿で算出（`estimateMinutesRemaining`）。最低観測窓60秒・充電中/非放電はnull・`trimSamples`で直近15分に整理・`formatDuration`で `3h07m` 表記。純ロジックでテスト9件。
- `App.tsx`：Battery Status API（Android/Chromium で利用可、iOS Safari は非対応のため自動的に非表示）を購読。ヘッダーに **バッテリーチップ** を表示——残量%、充電中は⚡、出力中は推定可動時間。残量15%以下かつ非充電で赤くパルス警告。

### マーカー打点の確認フィードバック
- マーカー記録時に画面下部へ **確認バッジ**（色ドット＋打点TC＋通算 `#n`）を約1.3秒フラッシュ表示。
- 端末対応時は **ハプティクス**（`navigator.vibrate(40)`）でも確認。画面を凝視せず打点を確信できる。
- タイマーはアンマウント時にクリーンアップ。

## 影響・検証

## 補足

iOS では Battery Status API が無いため、ネイティブ実装（Capacitor の電池情報）に置き換えるとiOSでも残量表示が可能。今回はWeb APIで依存追加なしに留めた。
