# ♻️ PC（デスクトップ）UIのスペース効率改善

- **日付**: 2026-06-29
- **種別**: 変更（UX / レイアウト）
- **主な対象ファイル**: `src/App.css`, `src/App.tsx`

## 背景

デスクトップの3カラム表示は、中央カラムにタイムコードカードが**240px固定**で配置され縦方向に大量の空白が生じていた。さらに各ペイン/フッターのpaddingが大きく、画面を有効に使えていなかった。

## 変更内容

### 中央カラムを使い切る大型タイムコード
- TCカードを固定高さ→**`flex: 1` で中央カラムの縦いっぱいに伸長**（`height:auto`）。
- キャンバスのフォント自動調整を**高さ基準も考慮**（`min(maxSize, 高さ×0.72)` → 幅で最終フィット）。デスクトップ上限を 220→**360px** に引き上げ、広い中央領域でTCが大きく表示されるように。モバイルは従来通り幅基準で不変。

### 密度の最適化（`.desktop-view` 上書き）
- グリッド列を `280/1fr/300` → **`252/1fr/284`**（中央を広く、サイドを引き締め）。
- サイドペインのpadding `32px 24px` → **16px**、中央 `16px 20px`、フッター `30/60/40` → **14/24px**。
- セクションラベル 0.8rem/10px → **0.72rem/6px**、tool-card 16/20 → **12/14**、入力 1.5rem → **1.25rem**、btn-pill/マーカーボタン/エクスポートボタンを縮小、`tab-pane`/`control-section` の gap を縮小。
- フッター圧縮分、マーカー一覧の高さを `100vh-400px` → **`100vh-300px`** に拡大。

## 影響・検証

- `tsc -b` 通過、ESLint クリーン、テスト **86件合格**、Viteビルド **64モジュール成功**。
- 変更はデスクトップ表示（`.desktop-view` / `.desktop-dashboard`）に限定。モバイルレイアウトとTCの見え方は不変。
- 上書きは末尾追記（同詳細度＋後勝ち）で実現し、既存ルールは温存。

## 補足

実機/ブラウザのデスクトップ幅でTCが中央を大きく占有し、左右の操作列がより密に収まることを目視確認するのが望ましい。
