# 🛠 停止=リセット/一時停止=保持 ＋ LTCロジック監査(BPCビット修正)

- **日付**: 2026-06-29
- **種別**: 修正 / 監査
- **主な対象ファイル**: `src/App.tsx`, `src/audio/ltcWorkletSource.ts`

## 1. 停止=リセット、一時停止=保持 の明確化

- `stopEngine(reset)` に引数を追加。
  - **STOP**（停止ボタンの長押し確定 / `handleStartStop`）→ `reset=true`：タイムコードを**開始基準へリセット**。
    - FREE → 設定した開始TC（`manualTimecode`）へ。
    - NETWORK（直近オフセット有り）→ NTP補正済みの現在時刻へ。
    - それ以外 → 端末時刻へ。
    停止後のアイドル表示が「止めた位置で凍結」ではなく、リセット値になる。
  - **PAUSE**（一時停止）→ `reset=false`：エンジンのタイムコードを**保持**し、RESUMEでその位置から継続。
- あわせて `handlePause` が `manualTimecode` を上書きしていた箇所を削除（FREEモードの開始TCを一時停止で壊さないように）。

## 2. LTCロジック監査

`LtcEngine.ts` と AudioWorklet（`ltcWorkletSource.ts`）を精査しました。

### 正しく実装されている点
- **バイフェーズマーク符号化**：各ビット境界で必ず反転、`1` のときのみビット中央で追加反転。境界をまたいで位相連続。✔
- **80ビットフレーム構成**：フレーム/秒/分/時の各桁（LSB先行）、ユーザービット8フィールド、ドロップフレームフラグ(bit10)、同期語 `0011111111111101`(bit64–79)。✔
- **ドロップフレーム**：分頭で0,1番を飛ばし、10分ごとは飛ばさない標準ルール。✔
- **有理FPSタイミング**：1フレームのサンプル数を `floor((fc+1)·sr·den/num) − floor(fc·sr·den/num)` で配分し、29.97/23.976 を正確な平均周期で生成。✔
- **ライブ設定反映**：音量/ユーザービット/出力モードは `config` メッセージで稼働中も反映。FPSは稼働中は変更不可（UIで無効化）で不整合なし。✔

### 発見した欠陥 → 修正
- **バイフェーズマーク極性補正ビット(BPC)未実装**（SMPTE 12M）。同期語を常に一意に識別させるための補正ビットが常に0だった。
  - 修正：80ビット中の0の個数が偶数になるよう、**bit27(25fps)/bit59(その他)** を設定。両ビットは元々未使用領域のため、タイムコードやユーザービットには影響しない。

### 軽微/設計メモ（今回は変更なし）
- ハードジャム時に位相が不連続になるが、LTCは極性非依存のため復号には無影響。
- `mono-r` 型は型定義のみでUI未使用（ワークレットは stereo 以外を mono-l 扱い）。

## 影響・検証

- `tsc -b` 通過、ESLint クリーン、テスト **86件合格**、Viteビルド **64モジュール成功**。
- 監査の結論：**LTC生成は概ね規格準拠で正常**。唯一の規格欠落（BPCビット）を補完。BPCの極性は規格上「0の個数を偶数化」で実装したが、実機デコーダでの最終確認を推奨。
