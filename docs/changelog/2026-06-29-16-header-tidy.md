# ♻️ 上部バー(ヘッダー)の整理・スマート化

- **日付**: 2026-06-29
- **種別**: 変更（UX / レイアウト）
- **主な対象ファイル**: `src/App.tsx`, `src/App.css`

## 背景

ヘッダー右側に「言語 / ? / 状態ピル / FPS·MODE / バッテリー」が一列に混在し、役割の異なる要素が並んで雑然と見え、狭い画面では詰まりやすかった。

## 変更内容

右側を**役割で2グループに分離**し、間に細いディバイダを置いて整理:

- **状態グループ** (`.hdr-status`)：状態ピル → FPS·MODE → バッテリー（情報の並び）。
- **ハイライン仕切り** (`.hdr-divider`)。
- **操作グループ** (`.hdr-actions`)：言語トグル＋ヘルプ(?)を、高さ24pxで揃えた小さなボタンとしてまとめて右端に配置。

並び順も「ユーティリティ→状態」から「状態→仕切り→操作」へ整え、視線の流れと役割分担を明確化。

- **モバイル**では冗長な **FPS·MODE メタを非表示**（FPSはタイムコードカード上に常時表示、同期モードはSYNCタブにあるため）。これでバーがすっきりし、横詰まり/はみ出しを解消。

## 影響・検証

- `tsc -b` 通過、ESLint クリーン、テスト **86件合格**、Viteビルド **64モジュール成功**。
- 言語不変は維持（言語トグル固定幅・状態ピル最小幅・各テキスト1行固定）。仕切り/操作グループは固定幅要素のみのため、言語切替で右端位置が動かない。

## 補足

実機で、ヘッダーが「状態 ｜ 操作」の2ブロックに整然と分かれて見えること、モバイルで詰まりがないことを確認するのが望ましい。
