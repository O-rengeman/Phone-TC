# Antigravity Dynamic Subagents 機能リファレンス

## 1. 概要
Antigravityにおける **Dynamic Subagents (動的サブエージェント)** は、メインエージェントが高度な目標を自律的に達成するため、特定の専門的役割を持つ子エージェント（サブエージェント）を動的に作成・実行する機能です。
メインエージェントのコンテキスト（対話履歴やコードベース情報）の肥大化を防ぎ、並行して別のスコープで作業を実行させることで、より高速かつ高品質な検証・開発を可能にします。

---

## 2. Browser Subagent (`browser_subagent`)
プロジェクト検証やフロントエンドデバッグで最も重要となるのが、ブラウザを自動操作する **Browser Subagent** です。

### 2-1. 主な機能と特徴
- **ブラウザの自律制御**: ローカルで稼働する Chrome ブラウザ等を起動し、クリック、キー入力、スクロール、ページの遷移などの操作を自律的に行います。
- **非同期実行と並行処理**: メインエージェントの処理スレッドとは別に非同期で動作するため、検証中にメインエージェントが次のプランを練るなどの並行作業が可能です。
- **自動レコーディングとスクリーンショット**: サブエージェントによるすべてのブラウザ操作は、自動的に WebP 動画形式で録画され、アーティファクトディレクトリに保存されます。また、重要な状態のスクリーンショットも取得できます。
- **ビジュアルフィードバック**: AIがブラウザを操作している間、ブラウザウィンドウの周囲には青い境界線が表示され、ユーザーの手動操作と競合しないよう視覚的に警告されます。

### 2-2. 基本的な動作フロー
1. **メインエージェントからの指示**:
   タスクの具体的な指示（例: 「ローカルサーバーを起動し、`/dashboard`にアクセスしてスイッチャーのTAKEボタンをクリックし、プログラム映像が切り替わるか検証せよ」）を添えて `browser_subagent` ツールを呼びします。
2. **実行とレコーディング**:
   サブエージェントがブラウザを開き、指示された手順を実行します。動作プロセスはすべて録画されます。
3. **結果のレポート**:
   処理完了後、サブエージェントは最終ステータス、DOMの状態、スクリーンショット、録画へのパスをメインエージェントに報告します。
4. **メインエージェントによる検証**:
   報告されたDOM状態やコンソールエラー、録画内容を確認し、機能が正常に動作しているか判定します。

## 3. その他のサブエージェント（Subagent-Driven Development）
実装プランの各タスクを独立して処理するために、専門のサブエージェントをディスパッチする手法も存在します。
- **Implementer (実装サブエージェント)**: 指定された機能の実装およびTDDによるテストの作成・実行を担当。
- **Spec Reviewer (仕様レビューサブエージェント)**: 実装されたコードが計画書（Spec）の要件を完全に満たしているか、余計なコードが含まれていないかを検証。
- **Code Quality Reviewer (品質レビューサブエージェント)**: リファクタリング観点や保守性、定数化などの観点からコード品質をレビュー。

---

## 4. デバッグにおける活用
本アプリ（Phone-TC）のような、WebRTCやシグナリング、リアルタイムのスイッチャー操作を伴うアプリケーションのデバッグにおいて、Browser Subagentは極めて有効です。
- **複数セッションのシミュレーション**: マスター画面とカメラ（クライアント）画面の双方をブラウザでエミュレートし、P2P接続やタイムコードの同期、タリーランプの連動が正しく行われるかをエンドツーエンドで確認できます。
- **UIトランジションの視覚的確認**: MIX（AUTO）トランジションのアニメーションやタリーの色変化が正しく描画されているかを、自動生成される WebP 動画で事後検証可能です。
