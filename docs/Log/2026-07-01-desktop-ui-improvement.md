# PC用UIのスペース効率・はみ出し改善

## 概要
PC（デスクトップ）表示時に発生していた、サイドパネル等のはみ出しや無駄なスペースを解消するため、`App.css` のレイアウトを調整しました。

## 対応内容
- **グリッド幅の調整**: `.desktop-dashboard` の3カラム幅（サイドメニュー）を少し広げ（例：左右それぞれを `320px` および `340px`）、コンテンツのはみ出しを防止しました。
- **パディングの最適化**: `.sync-pane` と `.tools-pane` のパディングを `32px 24px` から `20px 16px` に狭め、限られた画面内での情報密度を向上させました。
- **フッターの小型化**: フッター（`.fixed-footer`）の不要な余白を削減（パディングを `16px 32px 24px` に縮小）しました。また、メインアクションボタンのサイズも適度に縮小し、スペースを確保しました。
- **カードの余白削減**: `.tool-card` 内のパディングを `16px 20px` から `12px 16px` に縮小しました。
- **タイムコード表示部の高さ調整**: `.timecode-card-pro` の高さを `240px` から `180px` へ縮小し、縦方向のはみ出しを防止しました。
- **クライアント一覧の最適化**: `.clients-grid` について、デスクトップ表示時は `minmax(130px, 1fr)` を指定することで表示幅の余裕を持たせました。

これにより、ノートPCなどの画面でもスクロールやテキストのはみ出しが発生しにくくなりました。
