# 作業ログ: Tally Lamp 機能の実装 (Phase 2 - Phase 4)

## 日時
2026年6月30日

## 概要
`TALLY_PLAN.md` および `TALLY_DESIGN.md` に基づき、Tally 機能の基礎（Phase 1）に続き、ネットワーク越しの連動とトーチLED操作機能（Phase 2〜4）を実装しました。

## 実装内容詳細

### Phase 2: P2P全台一斉連動
- `src/utils/PeerSync.ts` の `SyncMessage` 型定義を拡張し、`tally`（`TallyPayload` 型）を追加。
- `src/App.tsx` に `tallyPayload` State を追加。
- Masterから送信される 10Hz の Heartbeat に Tally 状態を含めることで、ネットワーク瞬断時にも即座に復帰できるように実装。
- Client 側で、Tally Payload の `rev` を見て最新の Tally 状態を採用（`adoptTally`）するロジックを統合。

### Phase 3: カメラ個別指定 (P2P)
- `src/App.tsx` の Master 側 UI（CONNECTED CLIENTS 領域）に、端末ごと個別のタリー（LIVE / PREVIEW / STANDBY / OFF）を割り当てるボタンを追加。
- TallyPayload の `assignments` を活用して指定した Client に対して個別の Tally 状態が反映されるようにロジックを構成。
- AUTO モード時、Master の LTC 出力状態（`isRunning`）に連動して、TallyPayload の `all` を更新し自動で LIVE / STANDBY を切り替えるよう `useEffect` を実装。

### Phase 4: トーチ LED 操作機能
- Native 機能 (`TimecodeNativeBridge`) を拡張し、`setTorch({ on: boolean })` を追加。
- iOS: `AVCaptureDevice` を用い、安全にロックを取得して Torch モードを切り替える実装を追加 (`TimecodeNativeBridge.swift`)。
- Android: `CameraManager` を用い、背面カメラの ID を取得して Torch モードを切り替える実装を追加 (`TimecodeNativeBridgePlugin.java`)。
- Webフォールバック: `navigator.mediaDevices.getUserMedia` に `advanced: [{ torch: true }]` を渡すことでブラウザ上の対応も実装 (`App.tsx`)。
- UI に「Torch LED」のトグルを追加し、Tally State が `live` になった際に自動的にライトが点灯するように Effect を設定。
