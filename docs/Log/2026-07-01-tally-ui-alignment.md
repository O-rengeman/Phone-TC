# タリー画面とホーム画面の不整合（つじつまの合わない点）の改善作業ログ

**日付**: 2026-07-01  
**コミット**: `daba9b3` fix: resolve tally UI inconsistencies between home screen and fullscreen overlay

## 検出された不整合と改善内容

### 1. 手動タリー設定の選択肢の不整合（Toolsタブ）
- **不整合**: タリー画面とディレクター画面は「ON AIR / PREVIEW / OFF」の3択UIに変更されたが、ホーム画面（Toolsタブ）の自分向け手動タリー変更ボタンは `['live', 'standby', 'off']` のままで、PREVIEW（緑）が選べず、旧表示のSTBY（橙）が残っていた。
- **改善**: `['live', 'preview', 'off']` に変更し、UI上の選択肢を「ON AIR / PREVIEW / OFF」の3択に完全に統一。

### 2. 接続クライアント手動設定ボタンの不整合（Toolsタブ）
- **不整合**: Toolsタブ内の接続クライアント一覧にある手動切り替えボタンが、`['live', 'preview', 'standby', 'off']` の4択のままで、ディレクターパネルの3択仕様と不一致を起こしていた。
- **改善**: 同様に `['live', 'preview', 'off']` の3択に変更。

### 3. 非接続AUTOモード時の表示不整合
- **不整合**: 非接続（スタンドアロン）でAUTOモード動作時、LTCエンジンが停止していると内部状態 `standby` を返していた。この `standby` はプレビューにマッピングされたため、タイムコードが止まっている時（＝非録画時）も画面が緑色（PREVIEW）になってしまっていた。
- **改善**: `tally.ts` の `resolveTally` の仕様を修正。非接続時のAUTOモードは、LTC出力停止中であれば待機を意味する `off`（消灯）を返すようにロジックを修正。これにより「LTC停止中＝消灯、LTC動作中＝ON AIR（赤）」となり、実際の動作と視覚的フィードバックのつじつまが完全に合致した。
- **テスト**: `tally.test.ts` にある `resolveTally` のAUTOモード時のアサーション期待値を `off` に更新し、すべてのユニットテストがパスすることを確認。

### 4. 表示ラベルテキストおよびログテキストの直書き不整合（多言語化）
- **不整合**: タリー画面やディレクター画面で `ON AIR` などのテキストがJSX内に直接ハードコーディングされており、日本語切り替え時も英語表示のみになっていた。また、ログ内のステート表現が元の定義からずれていた。
- **改善**: 
  - `i18n.ts` を修正し、`tally.live` -> `ON AIR`、`tally.preview` -> `PREVIEW`、`tally.off` -> `OFF` とした上で、詳細な状態ラベル `tally.sub.live`（本番中）/ `tally.sub.preview`（次点・確認中）/ `tally.sub.off`（オフ）を多言語（日/英）で追加。
  - JSX内の表示をすべて `tr()` 翻訳関数経由に変更し、ホーム画面とタリー画面で一貫した多言語表示を実現。

## 検証とビルド結果
- `npm run build` は正常終了。
- `npx vitest run` を実行し、全98テストがすべて正常にパスすることを確認。
